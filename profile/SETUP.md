# How to activate your GitHub profile page (2 minutes)

GitHub shows a "homepage" README on your profile when you have a public repository named exactly the same as your username.

1. Go to <https://github.com/new>
2. Repository name: **Rahulprajapati99** (GitHub will show "✨ You found a secret!")
3. Set it to **Public**, check **Add a README file**, and click **Create repository**
4. Open the new repo's `README.md`, click the pencil (edit), delete the default content, and paste in the contents of [`profile/README.md`](./README.md) from this folder
5. Commit — your profile page at <https://github.com/Rahulprajapati99> now shows it

## Recommended follow-ups (5 more minutes, big impact)

- **Profile settings** (<https://github.com/settings/profile>): set your full name, bio ("QA Engineer × AI Builder — I design quality processes and build AI agents that run them"), location, and the portfolio link.
- **Pin repositories** on your profile: `Finance_Dashboard`, `automaton-earntisurvive`, and (once public) `Portfolio`.
- **Make `Portfolio` public** — recruiters can't see private repos, and a portfolio repo is exactly what they look for.
- **Review the Toolbox section** in the profile README and adjust the tool list to match what you actually use — accuracy beats volume in recruiter screens.
